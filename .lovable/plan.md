# Real YouTube ingestion

Replace the hardcoded `sample` lesson with a real pipeline: parse any YouTube URL, fetch metadata + transcript from free public sources, and derive a Lesson against the existing Zod schema. No AI yet — lesson content is templated from real transcript chunks so it actually reflects the video. AI generation is the next phase.

## Flow

```text
paste URL → /processing/$videoId → server fn fetches → /lesson/$videoId
              (oEmbed + transcript)        (real data, cached in memory)
```

## Pieces

**1. URL parsing (`src/lib/youtube.ts`)**
- `parseYouTubeId(input)` — accepts `youtube.com/watch?v=`, `youtu.be/`, `shorts/`, `embed/`, bare 11-char IDs. Returns `{ id } | null`.
- Landing page validates on submit; bad URL → inline error, no navigation.

**2. Server fn: metadata (`src/lib/ingest.functions.ts`)**
- `fetchVideoMeta({ youtubeId })` — calls `https://www.youtube.com/oembed?url=…&format=json` (free, no key). Returns `{ title, channel, thumbnail }`.
- Duration isn't in oEmbed → derive from transcript's last cue end (good enough for the UI).

**3. Server fn: transcript**
- `fetchTranscript({ youtubeId })` — tries free public sources in order:
  1. `https://www.youtube.com/watch?v=ID` HTML → extract `ytInitialPlayerResponse` → find `captions.playerCaptionsTracklistRenderer.captionTracks[*].baseUrl` (prefer English, then ASR) → fetch the `fmt=json3` variant → flatten to `{ start, dur, text }[]`.
  2. Fallback: same baseUrl without `fmt=json3` (XML) → parse with a tiny regex.
- All `fetch` calls run in the Worker SSR runtime (compatible). No npm package needed.
- Returns `{ cues: Cue[], language, isAutoGenerated }` or throws a typed error: `NO_CAPTIONS | PRIVATE_OR_BLOCKED | NOT_FOUND | UNKNOWN`.

**4. Lesson builder (`src/lib/buildLesson.ts`, pure)**
- Input: `meta + cues`. Output: a `Lesson` that passes the existing Zod schema.
- Segments: chunk transcript into ~6 equal slices, label them `skip|watch|core|demo|watch|skip` (deterministic placeholder until AI lands).
- Cards: pick 6 cues with the longest text from the middle 70 % of the video. Map to `concept / analogy / insight / quote / concept / recap` in order. `body` = the cue text (trimmed, ≤ 280 chars). `timestamp` = cue start.
- `bestPart` / `skipPart` = the `core` and first `skip` segment ranges.
- `watchScore` = `6 + min(4, cues.length / 200)` rounded to 1 dp.
- `keyMoments` = the 4 card timestamps.
- `quiz` = 3 templated questions referencing real card titles (single correct = "as stated in the video").
- `tutorSeed` = 3 generic Q/A that splice in the real title/channel/duration.
- `reallyAbout` / `scoreReason` / `recommendation` = templated strings that use real metadata. Clearly placeholder-ish wording so it's obvious AI will replace this.

**5. Server fn: full lesson**
- `getLessonByYoutubeId({ youtubeId })` — runs meta + transcript in parallel, calls `buildLesson`, returns the validated `Lesson`. In-memory `Map` cache keyed by `youtubeId` for the lifetime of the worker instance (cheap dedupe).

**6. Routing changes**
- Landing `/` submit: `parseYouTubeId` → `navigate({ to: '/processing/$videoId', params: { videoId: id } })` (the route param is now the real YouTube ID).
- `/processing/$videoId`: triggers `getLessonByYoutubeId` via `useServerFn` + `useQuery`. While loading, keeps the existing animated steps. On success → `navigate` to `/lesson/$videoId`. On typed error → friendly inline state with a "Try another URL" button (covers the "no captions" case explicitly).
- `/lesson/$videoId`, `/lesson/$videoId/player`, `/quiz`, `/done`: each route's loader calls `context.queryClient.ensureQueryData` for the same `getLessonByYoutubeId` query; components read with `useSuspenseQuery`. Removes all imports of `sampleLesson`.
- `sampleLesson.ts` stays as a dev fallback only — used when `videoId === 'sample'`.

**7. Cleanup**
- Update `Lesson.video.id` semantics: `id === youtubeId` now.
- `YouTubeEmbed` already takes `youtubeId` — no change.
- Empty/error states on `/processing/$videoId` use existing Tactile Field Guide tokens (brutal-border, brutal-shadow, mascot).

## Out of scope (next phase)

- Real AI-generated `reallyAbout`, cards, quiz, tutor answers (Lovable AI Gateway).
- Persisting lessons across sessions (needs Cloud + auth).
- Cross-video search / collections.
- Handling age-gated, members-only, or region-locked videos beyond a clean error message.

## Risks

- Some videos genuinely have no captions → surfaced as `NO_CAPTIONS` with a clear message; user can try another URL.
- YouTube occasionally changes the watch-page HTML shape. The extractor is small and easy to patch; failure is caught and surfaced as `UNKNOWN`.
- oEmbed and the watch page are public endpoints with no key, but unauthenticated; expect occasional rate limits per worker IP. Acceptable for the shell.
